轮询算法：FIFO + 获取后移到队尾

pool_acquire.go — tryAcquire（第66行）:

func (p *Pool) tryAcquire(exclude map[string]bool) (string, bool) {
    for _, id := range p.queue {       // 从头遍历 FIFO 队列
        if exclude[id] { continue }    // 跳过排除的账号
        if p.inUse[id] >= p.maxInflightPerAccount { continue } // 跳过已达并发上限的
        p.inUse[id]++                  // 占用一个并发槽
        p.bumpQueue(id)                // 移到队尾 → 其他账号向前推进
        return id, true
    }
    return "", false
}

pool_acquire.go — bumpQueue（第83行）:

func (p *Pool) bumpQueue(id string) {
    // 从当前位置移除
    for i, qid := range p.queue {
        if qid == id { p.queue = append(p.queue[:i], p.queue[i+1:]...); break }
    }
    // 追加到队尾
    p.queue = append(p.queue, id)
}

这不是传统的循环指针 Round-Robin，而是"获取后移到末尾"的自组织 FIFO。
效果等价于均匀轮询——刚用过的账号排到最后，其他账号依次前移。
